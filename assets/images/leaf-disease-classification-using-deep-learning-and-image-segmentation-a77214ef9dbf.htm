Part-1: Problem Definition and Objectives

Problem Definition:

  • Plants play a crucial role in people’s daily life in the form of vegetables, fruits and various consumer goods. So, identifying plant leaf diseases is very important for farmers in producing quality crops and also to enhance agricultural productivity. But, due to wide variety of diseases, leaf disease identification by human eye is time-consuming, difficult and also less accurate.
  • Therefore, there is a need for Automatic Leaf Disease Identification techniques which helps farmers to identify various diseases with less effort, less time and more accurate. These techniques also helps in healthy monitoring of fields that ensures quality agricultural products.
  • For this case study, different diseased-leaf images of soyabean plant are collected. Our task is to classify a given leaf among various diseases with high accuracy.

Objectives:

  • Predict the disease of the new leaf that is not yet identified.
  • Find the probabilities of leaf having various diseases so that the class with high probability value is chosen as predicted disease (Softmax).
  • Minimize the difference between the predicted and actual disease (Categorical Cross Entropy).

Evaluation Metric:

  • Accuracy is used as evaluation metric for this case study. It is defined as ratio of number of correctly predicted samples to the total number of predictions made.

Part-2: Data Collection

  • The dataset contains leaf images of soyabean plant. Data is collected from Digipathos Repository.
    Data Credits: https://www.digipathos-rep.cnptia.embrapa.br/jspui/
  • Only single leaf images are used in this case study. There are total 383 leaf images in this dataset.
  • This dataset has 10 different types of leaf diseases. They are Bacterial Blight, Brown Spot, Copper Phytotoxicity, Downy Mildew, Healthy, Mosaic Virus, Powdery Mildew, Rust, Rust & Target Spot and Southern Blight. Some of the leaf images are shown below:

Part-4: Leaf Image Segmentation

For extracting actual leaf pixels, we perform image segmentation using K-means clustering method as below:

Step 1: Finding the best color channel for color segmentation

  • Given raw images are in RGB color space. Due to high variance in colors of raw leaf images, it is difficult to apply k-means clustering directly on raw images.
  • So, we find best suitable color channel by splitting image in various available color spaces into individual channels.
  • After performing some operations, it is found that b channel of CIE Lab Color Space is best suitable for k-means clustering. For more details about Lab color space, visit https://en.wikipedia.org/wiki/CIELAB_color_space
# Trying different color spaces to find best color channel.image = cv2.imread('Soyabean-Original/BacterialBlight/IMG_0001.jpg')
img_rgb = cv2.cvtColor(image, cv2.COLOR_BGR2RGB)
img_lab = cv2.cvtColor(image, cv2.COLOR_BGR2LAB)
img_hsv = cv2.cvtColor(image, cv2.COLOR_BGR2HSV)# RGB Channel
R = img_rgb[:, :, 0]
G = img_rgb[:, :, 1]
B = img_rgb[:, :, 2]
fig, ax = plt.subplots(1,3, figsize=(15,15))

ax[2].imshow(B)

  • So, finally we choose b channel of Lab Color space for performing color segmentation using k-means clustering.

Step 2: K-means clustering on b channel of Lab color space

  • Since we want to extract only actual leaf pixels and remove other background pixels, we choose 2 clusters for k-means clustering.
  • One cluster for foreground leaf pixels where as other for unwanted background pixels.
# K-means clustering in opencv - https://opencv-python-tutroals.readthedocs.io/en/latest/py_tutorials/py_ml/py_kmeans/py_kmeans_opencv/py_kmeans_opencv.htmlpixel_vals = b.flatten()
pixel_vals = np.float32(pixel_vals)criteria = (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 10, 1.0)# Since we are interested in only actual leaf pixels, we choose 2 clusters
# one cluster for actual leaf pixels and other for unwanted background pixels.K = 2
retval, labels, centers = cv2.kmeans(pixel_vals, K, None, criteria, 10, cv2.KMEANS_RANDOM_CENTERS)
centers = np.uint8(centers)segmented_data = centers[labels.flatten()]
segmented_image = segmented_data.reshape((b.shape))
pixel_labels = labels.reshape(img_lab.shape[0], img_lab.shape[1])# displaying segmented image
plt.imshow(segmented_image)
  • From above image, it is observed that some of the unwanted background pixels are also clustered under actual foreground cluster.
  • In order to extract only actual foreground leaf pixels, we find all the connected components in foreground cluster and finally get the actual component containing leaf pixels.

Step 3: Finding connected components in 2-clustered pixel labels

  • Now, we find all the connected components among clustered pixels.
# Doing this, some unwanted pixels that are clustered in main cluster can be avoided.
# Ref - https://docs.opencv.org/3.4/d3/dc0/group__imgproc__shape.html#gac2718a64ade63475425558aa669a943apixel_labels = np.uint8(pixel_labels)
ret, components = cv2.connectedComponents(pixel_labels, connectivity=8)
plt.imshow(components, cmap='gray')

Step 4: Finding the main component to which actual leaf belongs

  • To find the main component in which actual pixels lie, simple logic is to find largest component among all connected components.
  • We filter largest component using max(no. of rows, no. of cols) in the component. Why not only no. of rows or no. of columns? We can also use this, but sometimes it may not work, because we have leaf images in both horizontal and vertical positions.
  • After obtaining main component, we create a mask for extracting actual pixels corresponding to main component
indices = []
for i in range(1, ret):
    row, col = np.where(components==i)
    indices.append(max(len(row), len(col)))component = np.argmax(np.array(indices))
main_component = component+1   #indexing starts from 0, so we increment by 1 to get actual component index# creating a mask and extracting pixels corresponding to cluster to which leaf belongs.
# 1 for actual leaf pixels and 0 for other pixels
mask = np.where(components==main_component, 1, 0)
B = image[:, :, 0]
G = image[:, :, 1]
R = image[:, :, 2]# Extract only masked pixels
r = R*mask
g = G*mask
b = B*maskfinal_img = np.dstack((r, g, b))
plt.imshow(final_img)
  • Now, we crop the segmented image such that all the surrounding black pixels are removed. Remove the pixels where pixel value is 0.
v_sum = np.sum(mask, axis=0)
h_sum = np.sum(mask, axis=1)w = np.count_nonzero(v_sum)
h = np.count_nonzero(h_sum)x_indices = np.where(v_sum != 0)
y_indices = np.where(h_sum != 0)x = x_indices[0][0]
y = y_indices[0][0]final_crop_img = final_img[y:y+h-1, x:x+w-1, :]
final_crop_img = np.uint8(final_crop_img)img = cv2.resize(final_crop_img, (640,480), interpolation=cv2.INTER_AREA)
plt.imshow(img)

For every dataset, we build two deep learning models with different architectures and compare their performances with evaluation metric (accuracy).

Data augmentation is performed on training data to ensure sufficient data for training.

For all deep learning models, Convolutional layers with various depths, Maxpooling, Dropout and BatchNormalization are used. For all hidden layers, relu is used as activation function where as softmax is used as activation function for output classification layer.

For every model, we save best model weights to the disk.

Part-6: Results and Conclusions

  • The results of all the deep learning models on 3 different datasets are:
  • From results table, it is observed that the performance of both CNN models on raw images is better than that of cropped and segmented images.
  • Since, dataset is very small and highly imbalanced, the performances of models are not best in terms of accuracy. If dataset is large, the performance of models might further improve.
  • Although image segmentation is performed on images, that has not helped in achieving better results than that of raw images.
  • In this case study, model-1 on original images is the winner among all the models.

“ A key lesson from this case study — Image segmentation might not always help deep learning models to give better results than without it. ”
